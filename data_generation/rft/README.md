# RFT Data Generation

This directory contains scripts to convert raw rollout data collected from experiments into the RFT (Robot Fine-Tuning) dataset format. The pipeline consists of three main steps: indexing, conversion, and metadata generation.

## Pipeline

### 1. Indexing (`generate_rft_index.py`)

Scans raw rollouts, identifies valid runs, and assigns unique episode IDs mapped to specific tasks.

**Usage:**

```bash
python data_generation/rft/generate_rft_index.py \
  <rollouts_root_dir> \
  <output_index_jsonl> \
  --tasks_jsonl <path_to_tasks_jsonl>
```

**Arguments:**

- `root_dir`: Root directory containing the rollout data.
- `output_jsonl`: Path where the generated index JSONL file will be saved.
- `--tasks_jsonl`: Path to the `tasks.jsonl` file containing task name to index mappings (default: `2025-challenge-demos/meta/tasks.jsonl`).

**Key Features:**

- Validates runs (checks for `state_action.npz`).
- Maps experiment directories to task names.
- Assigns incremental episode IDs, respecting existing indices in the output directory.

### 2. Conversion (`convert_rft_data.py`)

Processes the indexed rollouts into the final dataset format (Parquet state/actions, trimmed videos, etc.).

**Usage:**

```bash
python data_generation/rft/convert_rft_data.py \
  --input-root <rollouts_root_dir> \
  --output-root <output_dataset_root> \
  --jsonl <generated_index_jsonl> \
  --tasks-jsonl <path_to_tasks_jsonl> \
  --template-root <path_to_template_dataset>
```

**Arguments:**

- `--input-root`: The same root directory used in step 1.
- `--output-root`: Base directory for output. The script creates a subdirectory based on the index filename.
- `--jsonl`: The index file generated by step 1.
- `--tasks-jsonl`: Path to `tasks.jsonl`.
- `--template-root`: Path to an existing dataset (e.g., `2025-challenge-demos`) to copy meta/annotation templates from.
- `--num-workers`: Number of parallel workers (default: 32).

**Key Features:**

- Converts `state_action.npz` to Parquet.
- Trims initial static frames (default: 20).
- Updates metadata durations.
- Trims and re-encodes videos.

### 3. Metadata Generation (`generate_rft_meta.py`)

Generates dataset-level metadata files (`tasks.jsonl`, `episodes.jsonl`, `episodes_stats.jsonl`) required by the training pipeline.

**Usage:**

```bash
python data_generation/rft/generate_rft_meta.py \
  --data_dir <converted_dataset_dir> \
  --jsonl <generated_index_jsonl> \
  --tasks-jsonl <path_to_tasks_jsonl>
```

**Arguments:**

- `--data_dir`: The specific dataset directory created in step 2 (e.g., `<output_root>/<index_filename>`).
- `--jsonl`: The index file from step 1.
- `--tasks-jsonl`: Path to `tasks.jsonl`.

## Example Workflow

```bash
# 1. Generate Index
# Creates ./data/indices/batch1.jsonl
python data_generation/rft/generate_rft_index.py \
  ./data/raw_rollouts \
  ./data/indices/batch1.jsonl \
  --tasks_jsonl ./data/meta/tasks.jsonl

# 2. Convert Data
# Creates ./data/rft_dataset/batch1/ containing data/, meta/, videos/ etc.
python data_generation/rft/convert_rft_data.py \
  --input-root ./data/raw_rollouts \
  --output-root ./data/rft_dataset \
  --jsonl ./data/indices/batch1.jsonl \
  --tasks-jsonl ./data/meta/tasks.jsonl \
  --template-root ./data/2025-challenge-demos

# 3. Generate Metadata
# Updates ./data/rft_dataset/batch1/meta/ with aggregated info
python data_generation/rft/generate_rft_meta.py \
  --data_dir ./data/rft_dataset/batch1 \
  --jsonl ./data/indices/batch1.jsonl \
  --tasks-jsonl ./data/meta/tasks.jsonl
```
